# Contributing to Scotec.Revit

Thank you for your interest in contributing to the scotec-revit project! We welcome contributions from the community to help improve and extend the library.

---

## How to Contribute

### 1. Fork the Repository

- Click the "Fork" button at the top right of the [GitHub page](https://github.com/scotec-Software-Solutions-AB/scotec-revit).
- Clone your fork locally:

``` git clone https://github.com/<your-username>/scotec-revit.git ```

### 2. Create a Branch

- Create a new branch for your feature or bugfix:

```
git checkout -b feature/my-new-feature
```

- Push your branch to your fork:

´´´
git push origin feature/my-new-feature
´´´


### 5. Submit a Pull Request

- Go to your fork on GitHub and click "Compare & pull request".
- Provide a clear description of your changes and reference any related issues.

---

## Guidelines

- **Code Style:** Follow the existing code formatting and naming conventions.
- **Tests:** Add or update tests for new features and bug fixes.
- **Documentation:** Update documentation files (e.g., `README.md`, `RevitCommand.md`) if your changes affect usage or APIs.
- **Commit Messages:** Use clear, concise commit messages.
- **Issue Tracker:** If your contribution addresses an issue, reference it in your pull request.

---

## Reporting Issues

If you find a bug or have a feature request, please open an issue in the [GitHub Issue Tracker](https://github.com/scotec-Software-Solutions-AB/scotec-revit/issues) and provide as much detail as possible.

---

## Code of Conduct
